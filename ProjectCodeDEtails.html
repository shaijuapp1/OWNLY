      Purpose: Main app component, handles routing and authentication/authorization logic.

      Control Flow Explanation:
      
    
        
  1. State Initialization: user holds the Firebase Auth user (or undefined while loading). role holds the user's role from Firestore.
    2. 
        
  2. Auth Listener: useEffect sets up auth.onAuthStateChanged to listen for login/logout. When a user logs in, it fetches their role from Firestore using their email as the document ID. If the user document exists, role is set; otherwise, role is set to null and the user is redirected to /not-authorized.
    3. 
        
  3. Loading State: If the app is determining authentication or role (and not on /not-authorized), it shows Loading....
    4. 
        
  4. Routing: Uses react-router-dom to define routes:
          
      
            
    • /: If authenticated and role === "admin", shows Dashboard. If authenticated but not admin, redirects to /not-authorized. If not authenticated, shows Login.
      • 
            
    • /dashboard, /users, /banks: Only accessible if user is authenticated and role === "admin"; otherwise, redirects to /.
      • 
            
    • /not-authorized: Shows a message and a sign out button for unauthorized users.
      •

  5. Sign Out: The Not Authorized page provides a sign out button that logs the user out and redirects to the login page.
